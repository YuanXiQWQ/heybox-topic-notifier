<a href="https://heybox-topic-notifier.yuanxiqwq.deno.net/"><img src="https://deno.com/logo.svg" alt="Deno" width="16" height="16" align="absmiddle"> Heybox Topic Notifier</a>
— это лёгкое приложение на Deno для мониторинга публикаций в темах Heybox. Оно периодически читает реальные
публикации тем в соответствии с настройками каждой учётной записи, проверяет заголовки, содержимое,
комментарии и ответы по правилам ключевых слов, записывает совпадения в представления ожидания и истории,
а затем отправляет уведомления через настроенный канал.

## Возможности

- Панель управления: просмотр состояния опроса, общего числа совпадений, последнего совпадения и ожидающих
  совпадений, а также ручной запуск проверки
- Страница настроек: настройка ID тем, состояния включения, заметок, единицы интервала опроса, лимита
  публикаций, режима сортировки, языка интерфейса, тёмного режима и цвета темы
- Настройки учётной записи: регистрация, вход, выход, обновление имени пользователя и обновление пароля;
  данные учётных записей изолированы по ID пользователя
- Правила ключевых слов: поддержка общих правил, правил для отдельных тем, мест совпадения, учёта регистра
  и регулярных выражений
- Таблицы совпадений: ожидающие записи и записи истории поддерживают фильтры по диапазону времени,
  пагинацию, пакетное завершение и удаление
- Отладочные записи: имитация совпадений и тесты уведомлений, с серверными ограничениями частоты для
  ручного опроса и отладочных операций
- Каналы уведомлений: пользовательский Webhook, ServerChan, PushPlus, WxPusher, email API и SMTP
- Ретрансляция уведомлений: необязательный relay на Cloudflare Worker для PushPlus, WxPusher и ServerChan
- Безопасность: хэши паролей PBKDF2, сессии на базе KV, CSRF-токены, заголовки безопасности, журналы аудита,
  а также allowlist исходящих подключений с проверкой DNS

## Стек

- Deno 2 + TypeScript
- Hono
- Deno KV
- Deno.cron + локальный планировщик таймера
- HTML, отрисованный на сервере + чистый JavaScript/CSS
- Скрипт Cloudflare Workers для ретрансляции уведомлений

## Локальная разработка

Запустите сервер разработки:

```powershell
deno task dev
```

Затем откройте:

```text
http://localhost:8000
```

Чтобы переопределить значения по умолчанию, используйте `.env.example` как справочник и задайте
соответствующие переменные окружения в вашей среде выполнения. Зарегистрируйте учётную запись при первом
посещении; переменные окружения только задают начальные значения для новых учётных записей или данных по
умолчанию, а после этого страница настроек каждой учётной записи становится источником истины.

Приложение предоставляет страницы регистрации и входа. У каждой учётной записи изолированы настройки,
история совпадений, состояние опроса и конфигурация уведомлений, поэтому пользователи, использующие один
и тот же URL развёртывания, не делятся данными. Пароли пользователей хранятся в Deno KV как salted PBKDF2
хэши, а не в открытом виде. Сессии входа хранятся в Deno KV, а cookie браузера содержит только случайный
токен сессии. Изменения настроек, учётной записи и отладки проверяют CSRF-токен, а чувствительные операции
в публичных развёртываниях ограничиваются по частоте.

## Команды

```powershell
deno task dev
deno task start
deno task check
deno task clear-seen
```

`clear-seen` очищает маркеры обработанных публикаций, чтобы те же публикации можно было проверить снова;
используйте его в production с осторожностью.

## Развёртывание

См. [deployment](./deployment.md) для настройки Deno Deploy. Точка входа приложения задаётся секцией
`deploy` в `deno.json`; GitHub Actions только запускает проверки и не развёртывает приложение. Точка входа
развёртывания в `src/deploy.ts` объявляет Deno Deploy Cron, а фактический опрос выполняется только на
Production и на временной линии Git Branch `dev`. См. [worker](./worker.md) для настройки Worker ретрансляции
уведомлений.
