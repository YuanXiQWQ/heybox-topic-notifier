เป็นแอป Deno ขนาดเบาสำหรับตรวจสอบโพสต์หัวข้อของ Heybox แอปจะอ่านโพสต์หัวข้อจริงเป็นระยะ
ตามการตั้งค่าของแต่ละบัญชี ตรวจสอบชื่อเรื่อง เนื้อหา ความคิดเห็น และการตอบกลับกับกฎคำสำคัญ
บันทึกรายการที่ตรงกันในมุมมองรอดำเนินการและประวัติ แล้วส่งการแจ้งเตือนผ่านช่องทางที่กำหนดไว้

## คุณสมบัติ

- แดชบอร์ด: ดูสถานะการตรวจสอบ จำนวนรายการที่ตรงกันทั้งหมด รายการที่ตรงกันล่าสุด และรายการที่รอดำเนินการ
  พร้อมการตรวจสอบด้วยตนเอง
- หน้าการตั้งค่า: กำหนด ID หัวข้อ สถานะการเปิดใช้งาน หมายเหตุ หน่วยช่วงเวลาการตรวจสอบ จำนวนโพสต์สูงสุด
  โหมดการเรียงลำดับ ภาษา UI โหมดมืด และสีธีม
- การตั้งค่าบัญชี: สมัครสมาชิก เข้าสู่ระบบ ออกจากระบบ อัปเดตชื่อผู้ใช้ และอัปเดตรหัสผ่าน;
  ข้อมูลบัญชีถูกแยกตาม ID ผู้ใช้
- กฎคำสำคัญ: รองรับกฎที่ใช้ร่วมกัน กฎเฉพาะหัวข้อ ตำแหน่งการจับคู่ การแยกแยะตัวพิมพ์เล็ก/ใหญ่
  และนิพจน์ทั่วไป
- ตารางรายการที่ตรงกัน: รายการรอดำเนินการและรายการประวัติรองรับตัวกรองช่วงเวลา การแบ่งหน้า
  การทำให้เสร็จแบบกลุ่ม และการลบ
- รายการดีบัก: รายการที่ตรงกันจำลองและการทดสอบการแจ้งเตือน พร้อมการจำกัดอัตราฝั่งเซิร์ฟเวอร์สำหรับ
  การตรวจสอบด้วยตนเองและการดำเนินการดีบัก
- ช่องทางการแจ้งเตือน: Webhook แบบกำหนดเอง, ServerChan, PushPlus, WxPusher, email API และ SMTP
- รีเลย์การแจ้งเตือน: รีเลย์ Cloudflare Worker แบบเลือกใช้ได้สำหรับ PushPlus, WxPusher และ ServerChan
- ความปลอดภัย: แฮชรหัสผ่าน PBKDF2, เซสชันที่ใช้ KV, โทเคน CSRF, ส่วนหัวความปลอดภัย, บันทึก audit
  และ allowlist ขาออกพร้อมการตรวจสอบ DNS

## สแต็ก

- Deno 2 + TypeScript
- Hono
- Deno KV
- Deno.cron + ตัวจัดตารางเวลาท้องถิ่นแบบ timer
- HTML ที่เรนเดอร์ฝั่งเซิร์ฟเวอร์ + JavaScript/CSS แบบ vanilla
- สคริปต์ Cloudflare Workers สำหรับรีเลย์การแจ้งเตือน

## การพัฒนาในเครื่อง

เริ่มเซิร์ฟเวอร์สำหรับพัฒนา:

```powershell
deno task dev
```

จากนั้นเปิด:

```text
http://localhost:8000
```

หากต้องการแทนที่ค่าเริ่มต้น ให้ใช้ `.env.example` เป็นข้อมูลอ้างอิงและตั้งค่าตัวแปรสภาพแวดล้อมที่เกี่ยวข้อง
ใน runtime environment ของคุณ ลงทะเบียนบัญชีในการเข้าชมครั้งแรก; ตัวแปรสภาพแวดล้อมจะใช้เพื่อกำหนดค่าเริ่มต้น
ให้บัญชีใหม่หรือข้อมูลเริ่มต้นเท่านั้น หลังจากนั้นหน้าการตั้งค่าของแต่ละบัญชีจะเป็นแหล่งข้อมูลหลัก

แอปมีหน้าสมัครสมาชิกและเข้าสู่ระบบ แต่ละบัญชีมีการตั้งค่า ประวัติรายการที่ตรงกัน สถานะการตรวจสอบ
และการตั้งค่าการแจ้งเตือนแยกจากกัน ดังนั้นผู้ใช้ที่ใช้ URL การ deploy เดียวกันจะไม่แชร์ข้อมูลกัน
รหัสผ่านผู้ใช้ถูกจัดเก็บใน Deno KV เป็นแฮช PBKDF2 แบบใส่ salt ไม่ใช่ข้อความธรรมดา เซสชันเข้าสู่ระบบ
ถูกจัดเก็บใน Deno KV และ cookie ของเบราว์เซอร์มีเพียงโทเคนเซสชันแบบสุ่ม การเปลี่ยนแปลงการตั้งค่า
บัญชี และดีบักจะตรวจสอบโทเคน CSRF และการดำเนินการที่ละเอียดอ่อนจะถูกจำกัดอัตราใน deployment สาธารณะ

## คำสั่ง

```powershell
deno task dev
deno task start
deno task check
deno task clear-seen
```

`clear-seen` จะล้างเครื่องหมายของโพสต์ที่ประมวลผลแล้ว เพื่อให้สามารถตรวจสอบโพสต์เดิมซ้ำได้;
ใช้ด้วยความระมัดระวังใน production

## การ Deploy

ดู [deployment](./deployment.md) สำหรับการตั้งค่า Deno Deploy จุดเริ่มต้นของแอปถูกกำหนดโดยส่วน `deploy`
ใน `deno.json`; GitHub Actions จะเรียกใช้การตรวจสอบเท่านั้นและไม่ได้ deploy แอป จุดเริ่มต้นของการ deploy
ใน `src/deploy.ts` ประกาศ Deno Deploy Cron และการตรวจสอบจริงจะทำงานเฉพาะบน Production และ timeline ของ
Git Branch `dev` เท่านั้น ดู [worker](./worker.md) สำหรับการตั้งค่า Worker รีเลย์การแจ้งเตือน
